: The Image Embed element

A semantically perfect, HTML-valid, fully accessible tag depends on the kind of image you are embedding. But there is a gold-standard pattern.

Below are the correct versions for each scenario, including explanations.

Informative Image

                <img src="assets/img/bird-robin-cute-animals-940f8d.jpg" alt="A red robin" width="864" height="1024" decoding="async" loading="lazy" fetchpriority="low">

Why this is perfect:

  • alt: Describes the image meaningfully.

  • width/height: Prevents layout shift (CLS).

  • decoding="async": Allows the browser to decode without blocking.

  • loading="lazy": Improves performance when image is off-screen.

  • fetchpriority="low": Helps browser prioritize more important resources (optional).

  • No role attribute: Not needed for meaningful images.

Decorative Image

                <img src="assets/img/bird-robin-cute-animals-940f8d.jpg" alt="" role="presentation" width="300" height="300" decoding="async" loading="lazy" fetchpriority="low">

Why:

  • alt="" hides it from screen readers.

  • role="presentation" reinforces this (optional but recommended)
